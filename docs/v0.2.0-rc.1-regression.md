# JYM Log v0.2.0-rc.1 회귀 테스트

## 테스트 정보

- 테스트 버전: `v0.2.0-rc.1`
- 기준 브랜치: `main`
- 테스트 담당자: GangJ
- 주 테스트 기기: Android
- 테스트 환경:
  - PC Chrome
  - Android Chrome
  - Android 설치형 PWA
- 미검증 환경:
  - iPhone Safari
  - iOS 설치형 PWA

## 결과 표시

- `[x]` 통과
- `[ ]` 미실행
- `[!]` 오류 발견
- `[-]` 현재 환경에서 검증 불가

---

## 1. 앱 시작과 로그인

- [ ] 첫 화면이 빈 화면 없이 표시된다.
- [ ] Google 로그인 버튼이 정상 작동한다.
- [ ] 기존 로그인 상태가 유지된다.
- [ ] 로그아웃 후 다른 사용자 데이터가 표시되지 않는다.
- [ ] 로그인 후 동기화 상태가 정상적으로 표시된다.
- [ ] 설정 화면에 `v0.2.0-rc.1`이 표시된다.
- [ ] 콘솔에 빨간 오류가 없다.

## 2. 화면 이동

- [ ] 오늘·기록·분석·루틴·설정 탭이 작동한다.
- [ ] 좌측 상단 뒤로가기 버튼이 작동한다.
- [ ] Android 시스템 뒤로가기가 작동한다.
- [ ] 화면을 돌아왔을 때 이전 스크롤 위치가 복원된다.
- [ ] 운동 중 뒤로가기는 이탈 확인창을 먼저 표시한다.
- [ ] 모달이 열려 있을 때 뒤쪽 화면이 이동하지 않는다.

## 3. 설정과 테마

- [ ] 시스템·라이트·다크 테마를 선택할 수 있다.
- [ ] 선택한 테마가 앱 재실행 후 유지된다.
- [ ] 다크 모드에서 카드와 입력창이 흰색으로 튀지 않는다.
- [ ] 뒤로가기 버튼이 두 테마 모두 식별 가능하다.
- [ ] 테마 선택 행의 경계와 라운드가 어긋나지 않는다.
- [ ] 설정 스위치의 손잡이가 트랙 밖으로 나오지 않는다.
- [ ] Tab으로 설정 스위치에 접근할 수 있다.
- [ ] Space 한 번에 상태가 한 번만 변경된다.
- [ ] Enter 한 번에 상태가 한 번만 변경된다.

## 4. 루틴 관리

- [ ] 루틴을 새로 만들 수 있다.
- [ ] 루틴을 복제할 수 있다.
- [ ] 루틴 이름과 설명을 수정할 수 있다.
- [ ] 운동을 추가할 수 있다.
- [ ] 운동을 수정할 수 있다.
- [ ] 운동을 삭제할 수 있다.
- [ ] 운동 순서를 위·아래 버튼으로 변경할 수 있다.
- [ ] 모바일 드래그로 순서를 변경할 수 있다.
- [ ] 새로고침 후 루틴 변경 내용이 유지된다.
- [ ] 진행 중 운동이 있으면 위험한 루틴 변경이 차단된다.

## 5. 주간 일정과 오늘만 변경

- [ ] 요일별 루틴과 휴식일을 저장할 수 있다.
- [ ] 새로고침 후 일정이 유지된다.
- [ ] 오늘 지정된 루틴이 자동 선택된다.
- [ ] 오늘만 다른 루틴을 선택할 수 있다.
- [ ] 오늘만 변경을 해제할 수 있다.
- [ ] 운동 진행 중에는 자동 루틴 전환이 발생하지 않는다.

## 6. 운동 시작과 세트 입력

- [ ] 운동 시작 버튼이 작동한다.
- [ ] 운동 목록과 현재 운동 순서가 정확하다.
- [ ] 중량 증가·감소 버튼이 작동한다.
- [ ] 반복 증가·감소 버튼이 작동한다.
- [ ] 중량을 직접 입력할 수 있다.
- [ ] 반복을 직접 입력할 수 있다.
- [ ] 0보다 작은 중량이나 반복이 입력되지 않는다.
- [ ] 세트 완료와 완료 취소가 작동한다.
- [ ] 새로고침 후 세트 상태가 복원된다.

## 7. 휴식 타이머

- [ ] 세트 완료 후 타이머가 시작된다.
- [ ] 타이머가 1초씩 감소한다.
- [ ] 30초 추가 버튼이 작동한다.
- [ ] 종료 버튼이 작동한다.
- [ ] 앱을 백그라운드에 두었다 돌아와도 시간이 맞는다.
- [ ] 타이머가 모달 위로 겹쳐 보이지 않는다.

## 8. 운동 완료

- [ ] 운동 완료 버튼이 작동한다.
- [ ] 피로도 창이 표시된다.
- [ ] 피로도 1~5를 선택할 수 있다.
- [ ] 뒤로가기 시 피로도 창만 닫힌다.
- [ ] 기록 저장하고 완료 버튼이 작동한다.
- [ ] 저장 버튼 연속 입력이 차단된다.
- [ ] 완료 화면으로 이동한다.
- [ ] 완료 세트 수가 정확하다.
- [ ] 총 볼륨이 정확하다.
- [ ] 완료 운동이 기록 화면에 나타난다.

## 9. 기록과 분석

- [ ] 운동한 날짜가 달력에 표시된다.
- [ ] 이전 주와 다음 주 이동이 가능하다.
- [ ] 완료 기록 상세 화면이 열린다.
- [ ] 뒤로가기로 기록 목록에 돌아온다.
- [ ] 주간 운동 횟수가 정확하다.
- [ ] 주간 완료 세트 수가 정확하다.
- [ ] 주간 총 볼륨이 정확하다.
- [ ] 기록이 없을 때 오류 없이 빈 상태가 표시된다.

## 10. 동기화와 충돌

- [ ] 모바일 수정 내용이 PC에 반영된다.
- [ ] PC 수정 내용이 모바일에 반영된다.
- [ ] 오프라인 수정 내용이 로컬에 유지된다.
- [ ] 온라인 복구 후 대기 기록이 업로드된다.
- [ ] 동시 수정 시 충돌 안내가 표시된다.
- [ ] 이 기기 기록을 선택할 수 있다.
- [ ] 클라우드 기록을 선택할 수 있다.
- [ ] 충돌 처리 후 정상 동기화 상태로 돌아온다.
- [ ] 충돌 처리 중 버튼 중복 입력이 차단된다.

## 11. 모달과 접근성

- [ ] 운동 이탈 확인창의 배경·ESC·뒤로가기가 작동한다.
- [ ] 운동 편집창의 배경·ESC·X가 작동한다.
- [ ] 피로도 창의 배경·ESC·뒤로가기가 작동한다.
- [ ] 충돌창은 배경을 눌러도 닫히지 않는다.
- [ ] 모달이 닫히면 이전 요소로 포커스가 돌아간다.
- [ ] 모달 안에서 Tab 포커스가 밖으로 빠져나가지 않는다.
- [ ] 주요 버튼의 터치 영역이 충분하다.
- [ ] 피로도 선택 상태가 `aria-pressed`에 반영된다.

## 12. PWA와 업데이트

- [ ] Android 홈 화면에 설치할 수 있다.
- [ ] 설치형 PWA가 독립 화면으로 실행된다.
- [ ] 새 버전 배포 시 업데이트 안내가 표시된다.
- [ ] 나중에 버튼은 안내만 닫는다.
- [ ] 지금 업데이트는 한 번만 새로고침한다.
- [ ] 운동 진행 중에는 업데이트가 차단된다.
- [ ] 온라인 실행 후 비행기 모드에서도 기본 화면이 열린다.
- [ ] 이전 서비스 워커 캐시가 제거된다.
