# JYM Log 안정화 로드맵

## 문서 목적

이 문서는 JYM Log v0.2.0 개발 버전을 외부 테스트가 가능한 수준으로 안정화하기 위한 작업 기준을 기록한다.

새로운 기능보다 데이터 보호, 오류 방지, 테스트 가능성, Android 및 iOS 호환성을 우선한다.

---

## 현재 개발 상태

- 버전: `v0.2.0-dev-47`
- 배포: GitHub Pages
- 인증: Firebase Authentication
- 로그인 방식: Google OAuth
- 데이터베이스: Cloud Firestore
- 로컬 저장: LocalStorage
- 앱 형태: 모바일 중심 PWA
- 실제 테스트:
  - Android Chrome
  - Android 설치형 PWA
  - PC Chrome
- 추후 실기기 테스트:
  - iPhone Safari
  - iOS 홈 화면 PWA

## Android 테스트 기준

- Chrome과 설치형 PWA 모두 동작
- 로그인 유지
- 운동 상태 저장 및 복원
- 오프라인 입력 가능
- 온라인 복귀 후 동기화
- 드래그 순서 변경
- 숫자 키보드 정상 표시
- 하단 시스템 영역과 버튼이 겹치지 않음

---

## iOS 호환성 기준

실기기 검증 전에도 아래 기준을 코드에 반영한다.

- `safe-area-inset` 적용
- 입력창 글자 크기 최소 16px
- 터치 대상 최소 약 44px
- `100dvh`와 대체 높이 함께 사용
- Pointer Events 기반 입력 처리
- 팝업 인증 실패에 대한 안내 제공
- 키보드가 입력창과 저장 버튼을 가리지 않도록 구성
- Safari에서 지원 여부가 다른 API에 대체 동작 제공
- 앱 전환 및 화면 잠금 후 타이머 재계산
