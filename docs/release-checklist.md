# JYM Log v0.2.0 릴리스 체크리스트

## 릴리스 차단 기준

다음 중 하나라도 발생하면 정식 출시하지 않는다.

- 앱 초기화 실패 또는 빈 화면
- 로그인 실패
- 운동 기록 저장 실패
- 완료 기록 중복 생성
- 사용자 간 데이터 노출
- 동기화 과정의 기록 유실
- 주요 버튼 미작동
- PWA 업데이트 무한 새로고침
- 오프라인 실행 불가

## 버전과 캐시

- [ ] `config.js` 버전이 `v0.2.0-rc.1`이다.
- [ ] 설정 화면 버전이 동일하다.
- [ ] 서비스 워커 캐시 이름이 RC 버전과 일치한다.
- [ ] `index.html`과 `APP_SHELL`의 파일 주소가 일치한다.
- [ ] 수정한 파일에 이전 쿼리 문자열이 남아 있지 않다.
- [ ] 새 서비스 워커 적용 후 이전 캐시가 삭제된다.

## 코드 상태

- [ ] 브라우저 콘솔에 오류가 없다.
- [ ] 같은 JavaScript 파일이 중복 로드되지 않는다.
- [ ] 수정한 JavaScript 파일의 문법 오류가 없다.
- [ ] 모달별 ESC 이벤트가 중복 실행되지 않는다.
- [ ] 버튼 연속 입력이 중복 저장을 만들지 않는다.
- [ ] 사용자 입력값이 화면을 깨뜨리지 않는다.

## 데이터 보호

- [ ] 사용자 UID별 데이터가 분리된다.
- [ ] 로그아웃 시 다른 사용자 데이터가 남지 않는다.
- [ ] 진행 중 운동이 새로고침 후 복원된다.
- [ ] 완료 운동 세션이 한 번만 저장된다.
- [ ] 오프라인 대기 데이터가 네트워크 복구 후 전송된다.
- [ ] 충돌 해결 후 revision이 정상 증가한다.
- [ ] 루틴 수정으로 기존 완료 기록이 변경되지 않는다.

## 모바일 UI

- [ ] 360px 폭에서 가로 스크롤이 없다.
- [ ] 390~412px 폭에서 레이아웃이 깨지지 않는다.
- [ ] 하단 내비게이션이 시스템 영역과 겹치지 않는다.
- [ ] 키보드가 주요 저장 버튼을 가리지 않는다.
- [ ] 라이트와 다크 모드 모두 읽을 수 있다.
- [ ] 모달과 바텀시트가 화면 밖으로 넘치지 않는다.
- [ ] 주요 터치 대상이 약 44px 이상이다.

## PWA

- [ ] 일반 Chrome에서 실행된다.
- [ ] Android 설치형 PWA에서 실행된다.
- [ ] 오프라인 상태에서 앱 셸이 열린다.
- [ ] 업데이트 배너가 표시된다.
- [ ] 지금 업데이트가 한 번만 실행된다.
- [ ] 운동 중 강제 업데이트가 발생하지 않는다.

## 문서

- [ ] README의 현재 버전이 맞다.
- [ ] CHANGELOG에 RC 변경 사항이 있다.
- [ ] 알려진 문제 문서가 최신이다.
- [ ] 미검증 환경이 명시돼 있다.
- [ ] 회귀 테스트 결과가 기록돼 있다.

## 출시 결정

- [ ] P0 오류 없음
- [ ] P1 오류 수정 또는 해결 계획 기록
- [ ] Android Chrome 테스트 통과
- [ ] Android 설치형 PWA 테스트 통과
- [ ] 정식 릴리스 가능
