# Changelog

JYM Log의 버전별 주요 변경 사항을 기록합니다.

## [v0.3.0-dev.7] - 2026-07-23

### Added

- 공통 숫자 스크러버 컴포넌트
- 숫자 중앙 영역 좌우 드래그 조절
- 루틴 설정의 중량, 세트, 반복, 휴식 및 중량 간격 스크러버
- 운동 진행 화면의 세트별 중량·반복 스크러버
- 운동 종목 즐겨찾기
- 최근 저장한 운동 종목 최대 8개 표시
- 전체·즐겨찾기·최근 사용 빠른 보기
- 사용자 계정별 운동 선택 선호 정보 분리 저장

### Changed

- 운동별 중량 조절 간격을 운동 화면의 실제 증감 단위로 사용
- 수동 관리형에서도 중량 조절 간격을 직접 변경 가능
- 기존 중량·반복 버튼 처리 코드를 공통 숫자 스크러버로 통합
- 새 운동 추가 시 최근 사용 운동이 있으면 최근 탭을 우선 표시
- 운동 설정 저장 성공·실패 후 편집창 busy 상태를 항상 해제
- 운동 편집 HTML 구조와 피로도 모달 위치 수정
- 앱 버전을 v0.3.0-dev.7로 변경
- 정적 파일 캐시 쿼리를 dev0307로 통일
- 서비스워커 캐시를 v0.3.0-dev.7 전용 캐시로 변경

### Safety

- 스크러버 드래그 중에는 화면값만 미리 표시하고 조작 종료 시 저장
- 세로 이동이 우세하면 페이지 스크롤을 유지
- 운동을 선택하기만 하고 취소한 경우 최근 목록에 기록하지 않음
- 실제 운동 설정 저장이 완료된 경우에만 최근 목록 갱신
- 즐겨찾기와 최근 목록은 운동 기록 및 루틴 데이터와 별도로 저장
- 직접 입력 운동과 기존 완료 기록을 자동 변경하거나 삭제하지 않음
- 기존 운동 종료 검증과 완료 세션 중복 저장 방지 유지

## [v0.3.0-dev.6] - 2026-07-23

### Added

- 기본 운동 종목 카탈로그 35개
- 운동 이름 검색과 부위·추천 장비 필터
- 루틴 운동의 사용 장비 자유 선택
- 운동별 중량 조절 간격 직접 설정
- 카탈로그에 없는 사용자 운동 직접 입력
- 기존 운동 이름과 정확히 일치하는 템플릿 연결 제안
- 완료 세션의 운동 템플릿, 장비 및 주요 부위 메타데이터

### Changed

- 운동 종목과 루틴 내부 운동 식별자를 별도 데이터로 관리
- 루틴 카드에 사용 장비와 중량 조절 간격 표시
- 운동 설정 화면의 장비·부위 입력 HTML 구조 수정
- 앱 버전을 v0.3.0-dev.6으로 변경
- 정적 파일 캐시 쿼리를 dev0306으로 변경
- 서비스워커 캐시를 v0.3.0-dev.6 전용 캐시로 변경
- 운동 카탈로그 CSS를 오프라인 앱 셸에 포함

### Safety

- 기존 운동을 템플릿에 자동 연결하거나 이름을 임의로 변경하지 않음
- 이름 또는 별칭이 정확히 일치할 때만 연결을 제안
- 사용자가 종목 연결 후 저장해야 실제 루틴에 반영
- 일치하지 않는 기존 운동은 legacy 또는 custom 상태로 유지
- 기존 루틴 운동 ID와 완료 운동 기록은 삭제하거나 교체하지 않음
- 템플릿 추천과 관계없이 실제 사용 장비를 자유롭게 선택 가능

## [v0.3.0-dev.5] - 2026-07-23

### Added

- 운동 완료 전 공통 기록 검증
- 완료 세트가 없는 운동의 완료 및 Firestore 저장 차단
- 기존 완료 기록의 저장 합계와 실제 완료 세트 교차 검증
- 비정상 완료 기록 개수 앱 자가진단
- 동일 운동 세션의 동시 저장 요청 잠금

### Changed

- 완료 시각 또는 완료 세트가 없는 기존 기록을 기록 화면에서 제외
- 저장된 완료 세트 합계가 잘못된 경우 실제 세트 배열로 복구
- 앱 버전을 v0.3.0-dev.5로 변경
- 정적 파일 캐시 쿼리를 dev0305로 변경
- 서비스워커 캐시를 v0.3.0-dev.5 전용 캐시로 변경

### Safety

- UI, 운동 상태 및 Firestore 저장 계층에서 완료 상태를 단계별 검증
- 동일 세션 저장 중에는 진행 중인 저장 Promise를 재사용
- 세션 문서 ID는 운동 시작 시각을 기준으로 고정
- 기존 비정상 Firestore 문서는 자동 삭제하거나 덮어쓰지 않음
- 정상 완료 기록만 달력, 기록 목록 및 진행 분석에 사용

## [v0.3.0-dev.4] - 2026-07-22

- 최근 추정 1RM 기준 개인기록 갱신 표시
- 분석 운동 선택값의 브라우저 로컬 저장
- 운동 완료 후 종목 분석 자동 갱신

### Changed

- 벤치프레스 고정 분석을 모든 기록 운동의 선택형 분석으로 변경

- 정적 파일 캐시 쿼리를 dev0304로 변경
- 서비스워커 캐시를 v0.3.0-dev.4 전용 캐시로 변경

### Safety

- 완료된 운동 세트만 성과 분석에 포함
- 중량과 반복수가 모두 유효한 세트만 추정 1RM 계산
- 운동 기록 원본은 수정하지 않고 읽기 전용으로 분석
- 운동 이름, 중량, 반복 및 볼륨 외 개인정보는 분석 화면에 표시하지 않음

## [v0.3.0-dev.3] - 2026-07-22

- 최근 완료 세션 최대 200건 조회 제한 유지
- 증감률 계산 시 이전 기간이 0인 경우 0으로 나누지 않고 신규로 표시

### Fixed

- 기존 Progress Log 캐시가 계속 표시되던 문제
- 사용자별 운동 기록이 같은 LocalStorage 공간을 공유하던 문제
- 운동 완료 후 운동 시간이 계속 증가하던 문제
- 같은 완료 세션이 중복 문서로 생성될 가능성
- 오늘 운동일 표시 점이 글자색으로 표시되던 문제
- 운동 추가와 삭제 이벤트가 중복 등록될 수 있던 문제
- 운동 편집 초기화 코드가 중복 실행되던 문제
- 첫 번째와 마지막 운동의 순서 이동 범위 처리
- 운동 진행 중 루틴 구조가 변경될 수 있던 문제
- 운동 완료 버튼 연속 입력으로 완료 처리와 저장 요청이 반복될 수 있던 문제
- 완료 기록 저장 중 버튼이 다시 활성화될 수 있던 문제
- 오프라인 운동 변경 후 앱을 종료하면 미전송 상태가 사라질 수 있던 문제
- 클라우드 문서가 존재하면 더 최신인 로컬 상태도 덮어쓸 수 있던 문제
- 로그인 시 루틴을 불러오는 동작이 운동 상태 수정으로 기록될 수 있던 문제
- PC와 모바일이 동시에 운동 상태를 수정할 때 마지막 업로드가 다른 기기의 변경을 조용히 덮을 수 있던 문제
- 이 기기 기록 선택 중 클라우드 revision이 변경되면 충돌 해결이 반복되던 문제
- 실제 충돌 백업이 없는 임시 충돌 화면에서 기록 선택이 성공한 것처럼 표시되던 문제
- 충돌 모달 DOM 요소가 누락될 때 앱 실행 전체가 중단될 수 있던 문제
- 기기 시간대에 따라 오늘 루틴의 요일과 오늘만 변경 날짜가 어긋날 수 있던 문제
- Firestore 규칙에서 활성 루틴 설정과 주간 일정 문서 쓰기가 차단되던 문제
- 활성 루틴 설정 문서에 스키마 버전이 저장되지 않던 문제

### Known limitations

- PC와 모바일 동시 수정 시 충돌 해결 정책 미완성
- 자동 증량·유지·디로드 추천 미구현
