# Changelog

JYM Log의 버전별 주요 변경 사항을 기록합니다.

## [v0.2.0-rc.1.3.1] - 2026-07-21

### Fixed

- 진행 중 운동 복원 시 정의되지 않은 세트 검증 변수를 참조하던 문제
- 운동 UI 초기화 오류로 전체 버튼과 화면 이동이 작동하지 않던 문제
- 앱 버전과 업데이트 날짜가 표시되지 않던 문제

## [v0.2.0-rc.1.3] - 2026-07-21

### Fixed

- 초기 클라우드 조회가 응답하지 않을 때 동기화 상태가 계속 확인 중으로 남던 문제
- 설치형 PWA에서 이전 동기화 모듈이 재사용될 수 있던 캐시 주소 문제
- 동기화 초기화 예외가 화면 상태에 반영되지 않던 문제
- 세트 입력 change 이벤트가 중복 등록된 문제

## [v0.2.0-rc.1.2] - 2026-07-21

### Fixed

- 세트 중량과 반복 수에 음수를 직접 입력할 수 있던 문제
- 유효하지 않은 세트 입력값으로 완료 처리할 수 있던 문제
- 반복 수가 0인 세트에서 완료 버튼이 활성화되던 문제

## [v0.2.0-rc.1.1] - 2026-07-21

### Fixed

- 기본 루틴이 아닌 사용자 루틴에서도 삭제 버튼이 항상 비활성화되던 문제
- 루틴 삭제 가능 여부가 UI 공개 API에 전달되지 않던 문제

## [v0.2.0-rc.1] - 2026-07-21

### Added

- PWA 새 버전 감지와 사용자 선택형 업데이트 안내
- 공통 모달·바텀시트 레이어 관리자
- 모바일 접근성과 터치 피드백 보정
- 설정 스위치의 switch 역할과 접근성 상태
- 라이트·다크 테마 대비 보정
- v0.2.0 릴리스 회귀 테스트 및 체크리스트

### Changed

- 모달 표시, 포커스, 배경 스크롤, ESC 처리를 공통 관리 방식으로 통합
- 운동 중에는 PWA 업데이트로 인한 새로고침을 차단
- 설정 화면의 테마 선택과 우측 컨트롤 정렬 개선
- 설정 스위치의 터치 영역과 키보드 조작 개선
- 화면별 라이트·다크 색상 토큰 정리

### Fixed

- 운동 완료 피로도 창의 버튼이 눌리지 않던 문제
- 피로도 창에서 뒤로가기가 뒤쪽 화면에 적용되던 문제
- 운동 UI 스크립트가 중복 로드되던 문제
- 설정 스위치가 키보드 입력에서 두 번 실행될 수 있던 문제
- 다크 모드 뒤로가기 버튼과 설정 선택선의 대비 문제
- 토글 손잡이가 트랙 밖으로 보이던 문제

### Known limitations

- iPhone Safari와 iOS 홈 화면 PWA 실기기 테스트 미완료
- 운동 환경 설정 스위치는 실제 기능과 아직 연결되지 않음

## [v0.2.0-dev] - 2026-07-14

### Added

- Firebase Authentication 기반 Google OAuth 로그인
- 최초 로그인 사용자 자동 회원가입
- 브라우저와 설치형 PWA 자동 로그인
- 로그아웃 기능
- 사용자별 Firestore 프로필 자동 생성
- 사용자 닉네임 조회 및 수정
- 사용자별 LocalStorage 저장 공간 분리
- 진행 중 운동 상태 Firestore 동기화
- PC와 모바일 간 운동 상태 공유
- 클라우드 동기화 상태 표시
- 완료 운동 세션 Firestore 저장
- 실제 완료 기록 기반 운동 기록 화면
- 이전 주 이동이 가능한 주간 운동 달력
- 완료 운동 세션 상세 화면
- 실제 완료 기록 기반 주간 분석
- 사용자별 Firestore 운동 루틴
- 루틴 이름과 설명 편집
- 운동별 중량, 세트, 반복, 휴식, 증량 단위 편집
- 새 운동 추가 및 기존 운동 삭제
- 위·아래 버튼을 이용한 운동 순서 변경
- 마우스와 터치 드래그를 이용한 운동 순서 변경
- v0.2.0 안정화 로드맵
- 현재 운동 상태의 로컬 수정 시각 기록
- 사용자별 오프라인 동기화 대기열 영구 저장
- 앱 재실행 후 미전송 운동 상태 자동 복구
- 브라우저·PWA별 영구 기기 식별자
- 현재 운동 문서의 revision 기반 동시 수정 감지
- 동기화 충돌 시 로컬·클라우드 상태 별도 백업
- 동기화 충돌 상태 표시와 사용자 안내
- 복수 루틴 생성·복제·전환·삭제
- 주간 루틴 일정과 오늘 루틴 자동 선택
- 오늘만 루틴 변경과 일정 스냅샷 저장
- 완료 기록의 예정 루틴과 실제 수행 비교 표시

### Changed

- 운동 기록 저장 방식을 LocalStorage 단독 방식에서 LocalStorage와 Firestore 병행 방식으로 변경
- 로그인 사용자 UID를 기준으로 운동 기록과 루틴을 구분
- 고정된 예시 기록 화면을 실제 Firestore 완료 기록으로 교체
- 고정된 분석 수치를 실제 완료 운동 기반 계산으로 교체
- 코드에 고정돼 있던 기본 루틴을 사용자별 Firestore 루틴으로 이전
- 완료 운동에 루틴 정보와 운동별 세트 스냅샷을 저장하도록 변경
- 루틴 구조 변경 시 기존 진행 중 운동 초안을 안전하게 초기화
- 서비스 워커 앱 셸에 Firebase와 운동 관리 모듈 추가
- Android와 iOS 안전 영역을 고려하도록 주요 화면과 모달 개선
- 앱 시작 시 로컬과 Firestore 수정 시각을 비교해 최신 운동 상태를 선택하도록 변경
- 네트워크 복구 시 LocalStorage의 미전송 운동 상태를 다시 업로드하도록 변경
- 현재 운동 상태 저장을 Firestore 트랜잭션 기반 revision 검사 방식으로 변경
- 다중 기기 운동 기록 충돌 시 이 기기·클라우드 세트 값 비교 UI
- 동기화 충돌 선택창의 Escape 닫기, 포커스 복귀 및 처리 중 접근성 상태

### Fixed

- 기존 Progress Log 캐시가 계속 표시되던 문제
- 사용자별 운동 기록이 같은 LocalStorage 공간을 공유하던 문제
- 운동 완료 후 운동 시간이 계속 증가하던 문제
- 같은 완료 세션이 중복 문서로 생성될 가능성
- 오늘 운동일 표시 점이 글자색으로 표시되던 문제
- 운동 추가와 삭제 이벤트가 중복 등록될 수 있던 문제
- 운동 편집 초기화 코드가 중복 실행되던 문제
- 첫 번째와 마지막 운동의 순서 이동 범위 처리
- 운동 진행 중 루틴 구조가 변경될 수 있던 문제
- 운동 완료 버튼 연속 입력으로 완료 처리와 저장 요청이 반복될 수 있던 문제
- 완료 기록 저장 중 버튼이 다시 활성화될 수 있던 문제
- 오프라인 운동 변경 후 앱을 종료하면 미전송 상태가 사라질 수 있던 문제
- 클라우드 문서가 존재하면 더 최신인 로컬 상태도 덮어쓸 수 있던 문제
- 로그인 시 루틴을 불러오는 동작이 운동 상태 수정으로 기록될 수 있던 문제
- PC와 모바일이 동시에 운동 상태를 수정할 때 마지막 업로드가 다른 기기의 변경을 조용히 덮을 수 있던 문제
- 이 기기 기록 선택 중 클라우드 revision이 변경되면 충돌 해결이 반복되던 문제
- 실제 충돌 백업이 없는 임시 충돌 화면에서 기록 선택이 성공한 것처럼 표시되던 문제
- 충돌 모달 DOM 요소가 누락될 때 앱 실행 전체가 중단될 수 있던 문제
- 기기 시간대에 따라 오늘 루틴의 요일과 오늘만 변경 날짜가 어긋날 수 있던 문제
- Firestore 규칙에서 활성 루틴 설정과 주간 일정 문서 쓰기가 차단되던 문제
- 활성 루틴 설정 문서에 스키마 버전이 저장되지 않던 문제

### Known limitations

- 오프라인 상태에서 앱 종료 시 미전송 변경 보호 보완 필요
- PC와 모바일 동시 수정 시 충돌 해결 정책 미완성
- 자동 증량·유지·디로드 추천 미구현
- iPhone Safari와 iOS 홈 화면 PWA 실기기 테스트 미완료

## [v0.1.1] - 2026-07-13

### Added

- 설정 화면에 앱 이름, 버전, 업데이트 날짜 표시
- 현재 날짜 자동 표시
- iPhone 홈 화면 아이콘 설정
- 수동 회귀 테스트 문서

### Changed

- 앱 표시명을 Progress Log에서 JYM Log로 변경
- CSS를 `css/style.css`로 분리
- 앱 설정을 `js/config.js`로 분리
- 로컬 저장 기능을 `js/storage.js`로 분리
- 운동 상태와 계산 기능을 `js/workout.js`로 분리
- 화면 렌더링과 버튼 동작을 `js/app.js`로 분리
- 운동 상태 변경을 workout API를 통해 처리하도록 통일
- PWA Manifest를 JYM Log 기준으로 수정

### Fixed

- 이전 운동 기록 필드명 불일치 수정
- 운동 완료 후 기존 타이머 변수를 참조하던 문제 수정
- 이전 서비스 워커 캐시가 계속 표시되던 문제 수정
- 새 JavaScript와 CSS 파일이 오프라인 캐시에 포함되지 않던 문제 수정

- 최초 운동 기록 클릭형 프로토타입
- 오늘의 운동 화면
- 세트 기록
- 휴식 타이머
- 운동 완료 요약
- 기록, 분석, 루틴, 설정 예시 화면
- GitHub Pages 첫 배포
# v0.2.0-dev-48 (2026-07-20)

- 라이트 테마를 기준으로 유지하면서 동일 레이아웃의 다크 테마를 추가했습니다.
- 설정 화면에 `시스템 설정 따르기 / 라이트 / 다크` 선택을 추가하고 기기 로컬에 저장합니다.
- 시스템 테마 선택 시 운영체제의 색상 모드 변경을 즉시 반영합니다.
- 확정된 JYM LOG 방향을 기반으로 헤더 로고, 로그인 로고, 운동 카드 워터마크, PWA 아이콘을 통일했습니다.
- 하단 내비게이션 아이콘을 일관된 선형 스타일로 정리하고 루틴에는 바벨, 설정에는 단순 톱니바퀴를 적용했습니다.
- 홈과 설정 화면 제목을 한 줄로 정리하고 작은 화면에서 글자 크기가 자연스럽게 줄도록 보정했습니다.
